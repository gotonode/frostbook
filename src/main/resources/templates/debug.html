Debug

These features are so much fun! They make testing this platform a breeze.

The platform works normally (in "production mode") when these features are turned off. In actual, production-production mode, enabling debug-features will naturally not be possible.

What can I do with these?

Here's some neat things you can do with them:

  • Create a random profile (with or without admin privileges)
  • Autologin to any account from the search results
  • Populate the database (see below)
  • Make the current user popular (see below, below)
  • Reset the database, deleting all data but preserving database structure

Populating the database; what's the deal there?

When you populate the database, many things will happen:

  • We create an admin profile
  • We create 4 normal user profiles
  • The admin profile will befriend 2 of the normal users
  • The remaining 2 users will send a friend request to the admin
  • The admin will send a friend request to one profile
  • Users will send comments to the admin's profile page

How to become popular, quickly?

Once you make the current user popular, this will happen:

  • We create 5 random profiles and have them send you a friend request
  • We create 5 more random profiles, and INSTANTLY make them your friends

What's the password?

The password for randomly-created profiles is "debug". Creative, wouldn't you say?

But you don't need that password unless you're testing the login feature since you can autologin from the search results!

How to disable?

To disable these features, see the Debug-menu.
